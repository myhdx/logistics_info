# 钉钉物流查询应用需求说明

## 1. 概述

- **目标用户**：公司员工（销售、客服、财务等需要查询发货信息的人员）
- **核心问题**：ERP发货信息无法在钉钉移动端查看，查询物流状态需要登录多个系统，操作繁琐
- **期望结果**：在钉钉APP里就能查ERP发货信息和物流轨迹，查不到时能看到发货公司联系方式

## 2. 功能列表与优先级

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 钉钉免密登录 | 从钉钉工作台点击进入，自动获取用户身份 | P0 |
| 发货单列表 | 查询ERP发货单列表，支持搜索/筛选 | P0 |
| 发货单详情 | 查看具体发货单内容（商品、数量、金额等） | P0 |
| 物流轨迹查询 | 调用快递100查询物流轨迹 | P0 |
| 地图展示 | 在百度地图上显示货物当前位置 | P0 |
| 物流节点展示 | 显示发运节点流转信息（按时间倒序） | P0 |
| 查不到物流时展示发货方联系信息 | 显示发货公司名称、电话、地址，支持一键拨打 | P0 |
| 简化查询 | 语音输入 + 关键字模糊搜索 + 结果选择提示 | P0 |
| 物流知识库问答 | 基于知识库文档回答物流管理制度相关问题，支持多轮对话 | P1 |
| 物流推送通知 | 物流状态变更时钉钉推送 | P2（后续迭代） |

## 3. 用户故事与验收标准

### 故事1：钉钉免密登录
- **描述**：作为公司员工，我可以直接在钉钉工作台点击物流查询，免密登录应用
- **验收标准**：
  - 当用户从钉钉工作台点击进入时，应自动获取用户身份并跳转至首页显示用户姓名
  - 当授权失败时，应提示错误信息并允许重试

### 故事2：查看发货单列表
- **描述**：作为员工，我想查看我的发货单列表，以便快速找到需要的单据
- **验收标准**：
  - 当用户进入发货单页面时，应显示最近30天的发货单列表
  - 当用户输入单号/客户名时，应实时搜索并过滤列表
  - 当用户点击某条发货单时，应跳转至详情页
  - 当列表为空时，应显示"暂无发货单"的提示

### 故事3：查看发货单详情
- **描述**：作为员工，我想查看发货单的详细信息，以便了解具体发货内容
- **验收标准**：
  - 当用户查看详情时，应显示：单号、客户名、商品明细、数量、金额、发货日期、收货地址
  - 当存在关联物流单号时，应显示物流单号及"查看物流"按钮
  - 当无物流单号时，应显示"暂无物流信息"状态

### 故事4：查看物流轨迹
- **描述**：作为员工，我想查看物流运输轨迹，以便了解货物当前位置和运输状态
- **验收标准**：
  - 当用户点击"查看物流"时，应调用快递100查询物流轨迹
  - 当查询成功时，应在百度地图上标记当前位置，并在下方显示物流节点列表（按时间倒序）
  - 当查询失败（无轨迹）时，应显示"物流信息待揽收"或类似提示
  - 当物流单号无效时，应显示错误提示

### 故事5：查不到物流时展示发货方联系信息
- **描述**：作为员工，我想在查不到物流时能看到发货方联系方式，以便主动联系
- **验收标准**：
  - 当物流轨迹查询无结果时，应自动展示ERP中的发货方联系信息
  - 应显示：发货公司名称、联系电话、联系人、地址
  - 应提供"一键拨打"按钮，点击调起钉钉拨打电话

### 故事6：简化查询
- **描述**：可以使用语音方式，可以根据部分关键字进行查询，有相关提示用户进行选择
- **验收标准**：
  - 当用户输入部分关键字时，应支持模糊搜索并返回相关发货单列表
  - 当搜索结果多项时，应给出选择提示（如：显示"找到N个结果，请选择"）
  - 支持语音输入（调用钉钉ASR）并转换为文字进行查询
  - 示例：「查26号到上海的XX产品」→ 返回匹配订单的物流信息

### 故事7：相关物流管理/知识
- **描述**：可以问询相关物流管理制度和知识，对提供的文档内容进行对用户问询解答
- **验收标准**：
  - 能根据用户提问，从知识库中检索相关内容并回答
  - 支持多轮对话，能记住上下文（如：用户问"运单号是多少"，可基于前文回答）
  - 当知识库无相关内容时，应提示"暂无该信息，可联系管理员"

## 4. 非功能需求

- **性能**：列表页加载 < 2秒，物流查询 < 3秒
- **安全**：钉钉AccessToken需加密存储，接口需签名验证
- **兼容性**：钉钉iOS/Android最新版，H5页面
- **容错**：物流查询超时5秒后返回"查询超时，请稍后重试"
- **知识库**：支持导入PDF/Word/TXT格式的物流管理制度文档

## 5. 范围外与假设

- **本次不做**：
  - 物流状态变更推送（P2，后续迭代）
  - 订单导出/打印功能
  - 多语言切换

- **假设**：
  - 金蝶星翰ERP已配置好单据API（发货单、物流单）
  - 已有企业钉钉账号和管理员权限
  - 快递100已购买企业服务
  - 百度地图API已申请并配置
