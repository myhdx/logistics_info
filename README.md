# 钉钉物流查询应用

> 基于钉钉移动端的ERP发货信息与物流轨迹查询系统

## 项目概述

- **目标**：在钉钉APP里查询ERP发货信息和物流轨迹，查不到时显示发货方联系方式
- **技术栈**：uni-app + Spring Boot 3.x + MSSQL + PostgreSQL(pgvector) + AI
- **适配**：钉钉iOS/Android H5

## 功能模块

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 认证 | 钉钉免密登录 | P0 |
| 发货单 | 列表查询、详情、关键字搜索 | P0 |
| 物流 | 轨迹查询、地图展示、发货方联系信息 | P0 |
| 搜索 | 关键字模糊搜索、扫码查询 | P0 |
| 知识库 | AI对话问答（物流管理制度） | P1 |

## API 接口

### 认证
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/auth/login | POST | 钉钉免密登录 |
| /api/auth/refresh | POST | 刷新Token |
| /api/auth/logout | POST | 登出 |
| /api/auth/me | GET | 获取当前用户 |

### 发货单
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/delivery/list | GET | 发货单列表（分页+搜索） |
| /api/delivery/{id} | GET | 发货单详情 |
| /api/delivery/search | GET | 关键字模糊搜索 |

### 物流
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/logistics/query | GET | 查询物流轨迹 |
| /api/logistics/company | GET | 获取发货方联系信息 |

### 知识库
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/knowledge/import | POST | 导入文档 |
| /api/knowledge/search | GET | 向量检索 |
| /api/knowledge/chat | POST | AI对话 |

### ERP
| 接口 | 方法 | 说明 |
|------|------|------|
| /api/erp/delivery | GET | 查询金蝶发货单 |
| /api/erp/logistics/{no} | GET | 查询金蝶物流信息 |

## 前端页面

按园区设计规范开发（蓝色渐变、圆角卡片）：

- **首页** - 搜索、功能入口、最近发货单
- **发货单列表** - 分页列表、关键字搜索
- **发货单详情** - 详情信息、物流单号、一键拨打
- **物流轨迹** - 地图展示、时间线
- **知识库** - AI对话问答
- **我的** - 用户中心、功能菜单

## 配置说明

修改 `src/main/resources/application.yml`：

```yaml
# 数据库
spring:
  datasource: MSSQL配置
  datasource-postgres: PostgreSQL配置
  data.redis: Redis配置

# 第三方服务
dingtalk: 钉钉配置
kuaidi100: 快递100配置
kingdee: 金蝶ERP配置
llm: LLM配置
embedding: 向量服务配置
```

## 部署

### 环境要求
- JDK 17+
- PostgreSQL 15+ (带pgvector扩展)
- Redis 6.x
- Nginx 1.20+

### 打包
```bash
cd logistics_info
mvn clean package -DskipTests
```

### 运行
```bash
java -jar target/logistics-admin.jar
```
